## 이 문서는 누구를 위한 건가요?

이 문서는 아래 같은 상황을 위한 튜토리얼입니다.

- Swift 앱에서 리스트 데이터를 보여주고 싶다
- 운영자가 데이터를 쉽게 수정하길 원한다
- 그런데 서버는 만들고 싶지 않다

쉽게 말하면, **"엑셀처럼 데이터를 편집하면 앱에 자동으로 반영되면 좋겠다"** 라는 바람입니다.

여기서 질문이 생깁니다.

> "나는 진짜로 백엔드가 필요한가?"

<details>
<summary><strong>정답 보기</strong></summary>

항상 그런 것은 아닙니다. MVP, 해커톤, 개인 프로젝트, 읽기 전용 데이터라면 Google Sheets를 가벼운 데이터 소스처럼 활용할 수 있습니다.

비유하자면, 레스토랑을 차리지 않아도 푸드트럭으로 음식을 팔 수 있는 것과 비슷합니다. 규모가 작을 때는 가벼운 방법이 오히려 더 좋을 수 있어요.

</details>

## API가 뭐지?

아주 단순하게 말하면, **앱이 데이터를 주고받기 위해 사용하는 창구**입니다.

음식점에 비유하면 이렇습니다:

- 손님(앱)이 주문서를 카운터(API)에 넘기면
- 주방(서버)에서 음식(데이터)을 만들어서
- 카운터를 통해 손님에게 전달합니다

예를 들어 앱이

- 상품 목록을 가져오고
- 공지사항을 읽고
- 사용자 정보를 요청하는

통로를 보통 API라고 부릅니다.

이 저장소는 Google Sheets를 완전한 서버 API로 바꾸는 것은 아니지만, **앱이 읽을 수 있는 데이터 창구처럼 활용하는 방법**을 보여줍니다.

---

## 우리가 실제로 필요한 것

정리하면 우리에게 필요한 건 딱 세 가지입니다:

- 📝 데이터를 수정할 수 있는 곳 → Google Sheets
- 📱 앱에서 읽을 수 있는 방식 → gviz URL (다음 문서에서 배웁니다)
- ⚡ 빠르게 만들 수 있는 구조 → 서버 없이, 시트 하나로

> "이 3개를 가장 단순하게 만족하는 방법은 뭘까?"

<details>
<summary><strong>정답 보기</strong></summary>

Google Sheets를 데이터 저장소처럼 두고, Swift 앱에서 읽을 수 있는 간단한 데이터 창구처럼 사용하는 방법입니다.

서버를 직접 만들지 않아도, Google이 이미 만들어놓은 길을 빌려 쓰는 거예요.

</details>

## 구조를 먼저 생각해보기

보통은 이렇게 생각합니다.

```text
App → Backend → Database
     (서버)    (데이터베이스)
```

여기서는 이렇게 단순화합니다.

```text
App → Google Sheets
      (시트가 곧 데이터베이스!)
```

그런데 곧바로 질문이 생깁니다.

> "앱은 스프레드시트를 그대로 읽을 수 없는데?"

<details>
<summary><strong>정답 보기</strong></summary>

맞습니다. Google Sheets 원본 화면은 사람이 보는 용도이지, 앱이 읽기 쉬운 데이터 형식이 아닙니다.

비유하자면, 도서관에 책이 있어도 택배로 보내려면 포장이 필요한 것과 비슷합니다. 앱이 읽을 수 있는 형태로 "포장"해주는 특별한 주소가 있는데, 그게 바로 `gviz`입니다. 다음 문서에서 자세히 다룹니다.

</details>

## 먼저 가장 중요한 전제

> "시트를 만들기만 하면 앱에서 바로 읽을 수 있을까?"

<details>
<summary><strong>정답 보기</strong></summary>

아니요. 먼저 시트가 **공개 읽기 가능 상태**여야 합니다. 그래야 Swift 앱에서 로그인 없이 읽을 수 있습니다.

기본적으로 Google Sheets는 만든 사람만 볼 수 있게 되어 있어서, "누구나 볼 수 있게" 설정을 바꿔줘야 합니다.

</details>

### 해야 할 것

아래 단계를 순서대로 따라 해보세요:

**1단계: Google Sheets를 만듭니다** 📄

- [Google Sheets](https://sheets.google.com)에 접속해서 새 스프레드시트를 만듭니다.
- 이름은 자유롭게 지어도 됩니다. (예: "내 앱 데이터")

**2단계: 첫 번째 행을 header로 작성합니다** ✏️

- 첫 번째 행(1행)에 각 열의 이름을 적습니다.
- 이 이름들이 나중에 앱에서 데이터를 읽을 때 **키(key)** 가 됩니다.

예시 header:

```text
id | name | price
```

실제 시트에서는 A1에 `id`, B1에 `name`, C1에 `price`를 입력하면 됩니다.

**3단계: 공유 설정을 바꿉니다** 🔗

이 단계가 가장 중요합니다! 아래 순서대로 해보세요:

1. 시트 오른쪽 위의 **"공유"** 버튼을 클릭합니다.
2. "일반 액세스" 부분을 찾습니다. (기본값은 "제한됨"으로 되어 있습니다)
3. **"제한됨"을 클릭**해서 **"링크가 있는 모든 사용자"** 로 바꿉니다.
4. 역할은 **"뷰어(보기 가능)"** 로 설정합니다.
5. **"완료"** 를 클릭합니다.

> ⚠️ "편집자"가 아니라 **"뷰어"** 로 설정해야 합니다. 앱에서 읽기만 하면 되니까요.
